# 🎮 멀티플레이어 체스 완성!

## 🎉 완료 내용

로비 시스템과 플레이어 구분 기능이 추가되어 **진정한 멀티플레이 체스 게임**이 완성되었습니다!

---

## ✨ 주요 기능

### 1. 🚪 로비 시스템
- ✅ **게임 방 생성**: UUID 기반 고유 방 번호 자동 생성
- ✅ **방 입장**: 방 번호 입력으로 친구와 함께 플레이
- ✅ **자동 색상 할당**: 선착순으로 WHITE → BLACK 자동 할당
- ✅ **방 번호 복사**: 클립보드로 쉽게 공유

### 2. 🎨 플레이어 구분
- ✅ **내 색상 표시**: 상단 정보 패널에 내가 어떤 색인지 표시
- ✅ **턴 기반 권한**: 자기 턴에만 기물 이동 가능
- ✅ **상대 기물 잠금**: 상대방 기물은 선택조차 불가능
- ✅ **보드 회전**: 흑(BLACK) 플레이어는 보드가 180도 회전되어 표시

### 3. 🔄 실시간 동기화
- ✅ **WebSocket 통신**: SockJS + STOMP로 안정적인 실시간 통신
- ✅ **양방향 업데이트**: 한 플레이어의 이동이 상대방에게 즉시 반영
- ✅ **게임 상태 공유**: 체크, 체크메이트, 무승부 등 모든 상태 동기화

### 4. 🛡️ 보안 & 검증
- ✅ **세션 ID 인증**: 서버에서 각 플레이어를 고유하게 식별
- ✅ **서버 측 검증**: 턴 검증 및 체스 규칙 모두 서버에서 처리
- ✅ **치팅 방지**: 클라이언트 조작으로는 부정행위 불가능

---

## 🎮 사용 방법

### 플레이어 1 (방 만들기)
1. 웹 페이지 접속
2. **"새 게임 만들기"** 버튼 클릭
3. 자동으로 방이 생성되고 **WHITE 색상** 할당
4. 상단에 표시된 **방 번호**를 친구에게 공유
5. 친구가 입장할 때까지 대기
6. 게임 시작! (WHITE가 먼저)

### 플레이어 2 (방 입장하기)
1. 웹 페이지 접속
2. 친구에게 받은 **방 번호** 입력
3. **"입장하기"** 버튼 클릭
4. 자동으로 **BLACK 색상** 할당
5. 게임 시작! (WHITE 턴 대기)

## 🎯 UI/UX 특징

### 로비 화면
- 🌙 **다크 모드**: 눈에 편한 다크 테마
- ✨ **그라데이션 버튼**: 시각적으로 아름다운 인터페이스
- 📋 **원클릭 복사**: 방 번호를 클립보드로 바로 복사

### 게임 화면
- 📊 **정보 패널**: 방 번호, 내 색상, 현재 턴을 한눈에 확인
- 🎨 **색상 강조**: 현재 턴의 색상에 맞춰 보드 테두리 색상 변경
  - WHITE 턴: 황금색 (#ffb700)
  - BLACK 턴: 하늘색 (#a8c0ff)
- 🔄 **자동 회전**: BLACK 플레이어는 보드가 180도 회전
- ✨ **호버 효과**: 내 기물에 마우스를 올리면 살짝 확대
- 🎯 **이동 가이드**: 
  - 빈 칸: 부드러운 점 표시
  - 상대 기물: 강렬한 고리 표시

### 권한 관리
- 🔒 **턴 제한**: 내 턴이 아니면 아무것도 선택 불가
- 🚫 **색상 제한**: 상대방 기물은 클릭해도 반응 없음
- ⏰ **실시간 피드백**: 턴 변경 시 즉시 UI 업데이트

## 🏗️ 기술 스택

### 백엔드
- ✅ Spring Boot 4.0.1
- ✅ Spring WebSocket (STOMP)
- ✅ SockJS (폴백 지원)
- ✅ ConcurrentHashMap (멀티룸 관리)
- ✅ 세션 ID 기반 인증

### 프론트엔드
- ✅ 순수 JavaScript (프레임워크 없음)
- ✅ STOMP.js (WebSocket 클라이언트)
- ✅ 반응형 CSS3 (그라데이션, 애니메이션)
- ✅ 동적 DOM 조작

## 📡 WebSocket 플로우

```
1. 접속
   ├─ 페이지 로드
   ├─ WebSocket 연결 (/ws-chess)
   └─ 로비 화면 표시

2. 방 생성 (플레이어1)
   ├─ /app/create 전송
   ├─ 서버: UUID 생성
   ├─ /topic/create 수신
   ├─ /app/join/{roomId} 전송
   ├─ /user/queue/join 수신 (WHITE 할당)
   └─ 게임 화면 전환

3. 방 입장 (플레이어2)
   ├─ 방 번호 입력
   ├─ /app/join/{roomId} 전송
   ├─ /user/queue/join 수신 (BLACK 할당)
   └─ 게임 화면 전환

4. 게임 플레이
   ├─ 플레이어1: /app/move/{roomId} 전송
   ├─ 서버: 턴 검증 + 체스 로직 실행
   ├─ /topic/game/{roomId} 브로드캐스트
   ├─ 양쪽 플레이어 보드 업데이트
   └─ 턴 전환

5. 오류 처리
   ├─ 턴 위반: /topic/errors/{roomId}
   ├─ 규칙 위반: /topic/errors/{roomId}
   └─ 클라이언트: 알림 표시
```

## 🔧 실행 방법

### 서버 시작
```bash
./gradlew bootRun
```

### 접속
```
http://localhost:8080
```

### 테스트 (2개 브라우저 창)
1. **창 1**: 새 게임 만들기 → 방 번호 확인
2. **창 2**: 방 번호 입력 → 입장하기
3. **창 1**: WHITE 기물 이동 (e2 → e4)
4. **창 2**: BLACK 기물 이동 (e7 → e5)
5. 교대로 이동하며 게임 진행

## 🐛 알려진 제한사항

### 현재 미구현 기능
1. **이동 가능 위치 표시**: 현재는 모든 칸을 표시 (서버 API 필요)
2. **프로모션 타입 전송**: 프로모션 시 타입 선택이 서버로 전달되지 않음
3. **재연결 처리**: 새로고침 시 게임 복구 불가
4. **관전 모드**: 3명 이상 입장 불가
5. **타임아웃**: 제한 시간 없음

### 해결 방법
1. **이동 가능 위치**: `/movable?source={coord}` API 추가 필요
2. **프로모션**: MoveDTO에 `promotionType` 필드 추가
3. **재연결**: 세션 ID로 게임 복구 로직 추가
4. **관전**: GameRoom에 spectators 리스트 추가
5. **타이머**: Spring @Scheduled로 턴 타이머 구현
